# ISSUE: configLink 명시 시 user-maintained 보호 로직이 우선되어 sync 실패

## Metadata

```yaml
id: ISSUE-configlink-overrides-user-maintained-guard
status: issue
priority: high
component: pkg/workspacecli/sync_command.go
reported: 2026-03-03
```

---

## Summary

`workspace sync` 실행 시 `configLink`가 명시적으로 설정된 workspace에서
대상 경로에 user-maintained `.gz-git.yaml`이 이미 존재하면 에러가 발생하여
sync가 중단된다.

**에러 메시지:**
```
workspace 'mywork': config file /home/user/mywork/.gz-git.yaml exists
and is user-maintained; remove it manually to use configLink
```

---

## Reproduction

**설정 파일**: `~/devenv/workstation/.gz-git.yaml`
```yaml
workspaces:
  mywork:
    path: ~/mywork
    type: config
    configLink: ./gz-git/mywork-gz-git.yaml   # ← 명시적으로 설정됨
    sync:
      recursive: true
```

**환경**: `~/mywork/.gz-git.yaml` 이 일반 파일(regular file)로 이미 존재.
파일 내부에 `Generated by gz-git` 마커 없음 → user-maintained 판정.

**재현 명령:**
```bash
cd ~/devenv/workstation
gz-git workspace sync --dry-run
```

---

## Root Cause Analysis

### 흐름

1. `ensureChildConfigs` (sync_command.go:1279~1295) 에서:
   - `ws.ConfigLink != ""` 분기 진입 (configLink 설정 감지)
   - `childConfigFile = filepath.Join(wsPath, ".gz-git.yaml")`
   - `IsConfigSymlink(childConfigFile)` → `false` (일반 파일)
   - 파일 내용에 `AutoGeneratedMarker` 없음 → user-maintained 판정
   - **에러 반환** → sync 전체 중단

2. 코드 위치: `sync_command.go:1282~1295`

```go
// FILE: pkg/workspacecli/sync_command.go:1279
if ws.ConfigLink != "" {
    childConfigFile := filepath.Join(wsPath, ".gz-git.yaml")
    isSymlink, _ := config.IsConfigSymlink(childConfigFile)
    if !isSymlink {
        if content, err := os.ReadFile(childConfigFile); err == nil {
            header := string(content)
            if len(header) > 500 {
                header = header[:500]
            }
            isGenerated := strings.Contains(header, "Generated by gz-git") ||
                strings.Contains(header, config.AutoGeneratedMarker)
            if !isGenerated {
                // ← 여기서 에러 발생. configLink 명시에도 불구하고 중단.
                return fmt.Errorf("workspace '%s': config file %s exists and "+
                    "is user-maintained; remove it manually to use configLink",
                    name, childConfigFile)
            }
        }
    }
    // ...
}
```

### 문제

**`configLink`를 명시적으로 설정한 것 자체가 "이 파일을 symlink로 교체하겠다"는
사용자 의도의 표명**이다. 그런데 기존 파일의 user-maintained 여부를 확인하여
에러로 차단하는 것은 사용자 의도와 역행한다.

보호 로직의 원래 목적은 **gz-git이 자동 생성한 파일을 실수로 symlink로 바꾸는 것을
방지**하는 것이었으나, `configLink`가 명시된 경우에는 이 로직이 불필요하다.

---

## Expected Behavior

`configLink`가 명시된 경우:
1. 기존 `.gz-git.yaml`이 일반 파일이어도 **경고(warn)만 출력**하고 진행
2. 기존 파일을 backup 후 symlink 생성 (또는 그냥 교체)
3. 최소한 `--force` 플래그 없이도 진행 가능해야 함

또는 다음 중 하나를 선택:
- **A안 (Warn & Replace)**: 경고를 출력하고 symlink로 교체. `configLink` 명시 = 명시적 동의.
- **B안 (Warn & Skip)**: 경고만 출력하고 이미 존재하는 파일 유지 (configLink 무시). sync는 계속.
- **C안 (--force 플래그)**: `--force` 또는 `--allow-overwrite` 플래그로 교체 허용.

> **권장**: **A안** - `configLink`가 명시적으로 설정된 것은 사용자의 명확한 의도이므로,
> 경고 후 교체가 최적. `CreateConfigSymlinkForce` 함수가 이미 존재하므로 구현도 단순.

## Proposed Fix

### 수정 위치: `pkg/workspacecli/sync_command.go:1279~1300`

```go
// BEFORE (현재 코드 - 에러 발생)
if ws.ConfigLink != "" {
    childConfigFile := filepath.Join(wsPath, ".gz-git.yaml")
    isSymlink, _ := config.IsConfigSymlink(childConfigFile)
    if !isSymlink {
        if content, err := os.ReadFile(childConfigFile); err == nil {
            // ... auto-generated 체크 ...
            if !isGenerated {
                return fmt.Errorf(...) // ← 에러로 중단
            }
        }
    }
    fmt.Fprintf(out, "→ Linking workspace '%s': %s → %s\n", ...)
    if err := config.CreateConfigSymlinkForce(ws.ConfigLink, wsPath, configDir); err != nil {
        return fmt.Errorf(...)
    }
    continue
}
```

```go
// AFTER (제안 - 경고 후 교체)
if ws.ConfigLink != "" {
    childConfigFile := filepath.Join(wsPath, ".gz-git.yaml")
    isSymlink, _ := config.IsConfigSymlink(childConfigFile)
    if !isSymlink {
        if _, statErr := os.Stat(childConfigFile); statErr == nil {
            // configLink가 명시되어 있으면 기존 파일이 user-maintained여도
            // 경고만 출력하고 symlink로 교체한다.
            // (configLink 명시 = 사용자의 명시적 의도)
            fmt.Fprintf(out,
                "  ⚠ workspace '%s': replacing existing config with configLink symlink: %s\n",
                name, childConfigFile)
        }
    }
    fmt.Fprintf(out, "→ Linking workspace '%s': %s → %s\n",
        name, childConfigFile, ws.ConfigLink)
    if err := config.CreateConfigSymlinkForce(ws.ConfigLink, wsPath, configDir); err != nil {
        return fmt.Errorf("failed to create config symlink for workspace '%s': %w", name, err)
    }
    continue
}
```
